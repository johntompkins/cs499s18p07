"Cold" Installation

  1. Prerequisites: Have a server running Linux Apache PHP MySQL stack. Python 3 is required as well. Also need a Google Cloud Account with Cloud vision and Cloud Language enabled.
  2. In Google Cloud, Create a project, enable the Vision and Language API for said product, and obtain a service account key following this guide: Google Credentials.
    • This should result in a JSON file being downloaded to your machine
    • Transfer this file to your server, and store it in a secure place on your server
    • Copy the path of this JSON file, and store the path in an environment variable called GOOGLE_APPLICATION_CREDENTIALS
  3. Copy contents of application zip file to directory /var/www/html/cs499SupremeCourt/
  4. Using SCDB.sql file contained in application files, import this into MySQL
  5. Python Packages: The following need to be installed using the following command: pip3 install PACKAGE_NAME
      newspaper3k
      beautifulsoup4
      google-cloud
      pillow
      newsapi-python
      nltk
  6. For the application to perform language analysis, in python, one must import and download a specific library with the following statements:
    • import nltk
    • nltk.download("punkt")
  7. Automating script
    1. Script is automated using crontab, which allows for the regular execution of shell commands.
    2. The shell command we need to execute regularly is "python3 /var/www/html/cs499SupremeCourt/source/article_collector/article_collector.py"
    3. To edit tasks completed regularly, we need to run command "crontab -e".This opens an editor that we can use to add a job.
    4. In editor, add line: "* * 0 0 0 python3 /var/www/html/cs499SupremeCourt/source/article_collector/article_collector.py" and save. This runs the collector daily at midnight.
    5. More information on how to edit the frequency of jobs can be found here: Crontab FAQ
  8. With the script automated, libraries installed, and files in the right place on the server, one should be able to access the site from http://yourDomainOrIP/cs499SupremeCourt/webapp/

AWS Installation

  1. To install for our customers, we have decided to take advantage of AWS's Amazon Machine Image (AMI) feature that allows one to take a snapshot of a server and all of its files.
  2. We have a private AMI saved of our application, and will be able to transfer this to the customers whenever we get their personal AWS accounts set up
  3. For non technical clients, this requires a lot fewer steps than a whole LAMP installation.
  4. The process for doing this with a shared AMI is described here in detail.
  5. The instance should be acessible from anywhere on port 80. This is configurable in the settings.
