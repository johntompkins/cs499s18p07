Testing

1. Unit Testing

  • Article Collector
    • Dedicated websites scrapers. Each scraper function will return articles from the link given in the function of the specific website. When run, this will return a list of urls of the supreme court articles
    • NEWS API function - This function returns a list of urls of articles that are relevant to the query "Supreme Court" from the NEWS API service.
    • RSS feeds - Given RSS feeds will be parsed for articles.
    • Relevancy - The articles from the RSS feeds and NEWS API will be tested for relevancy
    • Article Parsing - Given the urls from the various sources, this parsing ensures that the article title, images, text, source, and authors are lifted from the url. For each article, the collector should be able to lift this information.
    • Article Duplication Test - Given an article, check the database to determine if it is a duplicate.
    • Text analysis - Given an article text, send a request to Google Natural Language Service to determine keywords.
    • Image analysis - Given an image, send a request to Google Cloud Image analysis to determine entities in said image.
  • Web application
    • Form submission functionality - Make sure the keywords user submits are correctly handled and sent to the php script to retrieve information from the database.
    • Also make sure the date submissions are valid and in the correct format when sent to the script.

3. System Testing

  • Compatibility Testing
    • Front end web app - Browser - Chrome
    • Web server - AWS ec2 Container with LAMP STACK. Need to ensure all components of application function when it is moved to a university AWS account as they did on the development instance that we are using now.
    • Article Collector - AWS ec2 Container. Need to ensure that the collector functions correctly on Python 3.6 on the instance.
  • Ensure that the article collector is runnning on a regular basis. Collector should run at a predetermined interval.

4. Customer/User Testing

  • Test run from customers
    • Have the customers go through a test run so that they get an idea of how the program functions. They will be able to focus on specific test scenarios they find crucial for the funcionality of the program.
  • Boundary/extreme testing
    • Test the limits of all the search functions. The boundaries for the dates selected; maximum/minimum keyword length; maximum/minimum keywords
