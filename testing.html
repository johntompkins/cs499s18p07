1. Unit Testing

  • Article Collector
    • Website-specific scraper functions
      • Test that each scraper function can connect to its given website
      • Test that each scraper function successfully parses its given website
    • NEWS API function
      • Test that NEWS API key is correct and allows authentication
      • Test that NEWS API successfully pulls articles with keyword "United States Supreme Court"
    • RSS feeds
      • Test that Google Alerts account associated with feed is still active
      • Test that RSS feeds are on relevant search queries ('SCOTUS', 'Supreme Court', etc.)
      • Test that article collector can successfully connect to the feeds
    • Article Parsing
      • Test that article is successfully downloaded from URL
      • Test that article's text is successfully parsed
        • Test that authors are retrieved correctly
        • Test that article body is retrieved correctly
        • Test that publish date is retreived correctly, if available
        • Test that article title is retreived correctly
    • Relevancy analysis
      • Test that articles pertaining to state or foreign supreme courts are excluded
      • Test that avoided sources are successfully avoided
    • Redundancy Check
      • Test that duplicate articles are excluded from the database
    • Text analysis
      • Test that article collector successfully connects to Google Natural Language Service
      • Test that Google Natural Language Service correctly returns keywords from given articles
    • Image analysis
      • Test that article collector successfully connects to Google Vision Image analysis
      • Test that Google Vision Image analysis correctly returns keywords pertaining to given images
  • Web application
    • Form submission functionality
      • Test that the keywords user submits are correctly handled and sent to the php script to retrieve information from the database.
      • Test that the date submissions are valid and in the correct format when sent to the script.
    • Display functionality
      • Check that all sources are correctly listed and selectable
      • Check that articles' contents are displayed correctly
      • Check text analysis keywords are displayed correctly

2. Integration Testing

  • Synchronization Testing
    • Test that article collector can read and write to back-end SQL database
    • Test that web display can connect to SQL database
    • Test that articles are retreieved correctly from backend SQL
    • Test that an appropriate message displays if database is unavailable
  • Function Testing
    • Test that the URLs returned by NEWS API are properly returned to the parsing function.
    • Test that the RSS feeds can be properly processed by the series of functions.
  • Boundary/extreme testing
    • Test the limits of all the search functions. The boundaries for the dates selected; maximum/minimum keyword length; maximum/minimum keywords
    • Test articles that have abnormally large amounts of test and/or images, ensuring everything is prominently presented to the customer on-screen.

3. System Testing

  • Compatibility Testing
    • Test that front end shows properly on all major browsers (Microsoft Edge, Chrome, Firefox, Safari, ...)
    • Test that article collector runs properly; all libraries are installed, and can be run periodically (once a day, for example)
    • Test that Amazon ec2 server is reachable from public internet

4. Customer/User Testing

  • Test run from customers
    • Have the customers go through a test run so that they get an idea of how the program functions. They will be able to focus on specific test scenarios they find crucial for the funcionality of the program.
